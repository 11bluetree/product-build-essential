# Security Baseline

## 最低限守ること

- すべての外部入力を検証する
- 認証が必要な操作は毎回検証する
- 権限チェックを明示する
- 状態変更系の操作は CSRF や同等の保護を行う
- SQL や検索条件の組み立ては必ず安全な方法を使う
- 秘密情報や決済情報をログへ出さない

## 設計上の原則

- セキュリティ要件を後付けにしない
- 認証方式、セッション方式、トークン方式は抽象化の背後に置く
- 外部サービスへ責務委譲できるものは委譲する
- 機密データは保存前、送信前、表示前で扱いを分ける

## レビュー観点

- 入力検証抜けがないか
- 権限の境界が曖昧でないか
- エラーメッセージに過剰な内部情報が含まれていないか
- 環境変数の公開範囲が整理されているか
