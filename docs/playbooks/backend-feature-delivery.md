# Backend Feature Delivery

## 実装順序

新しいバックエンド機能は、原則として以下の順序で進める。

1. ドメインの振る舞いを定義する
2. ユースケースを定義する
3. 外部依存のインターフェースを決める
4. インフラ実装を作る
5. API 境界を作る
6. 統合テストを追加する

## 進め方

### 1. ドメイン

- エンティティ、値オブジェクト、集約を定義する
- 不変条件と失敗条件を決める
- 業務ルールをテストで先に固定する

### 2. ユースケース

- 1 つのユーザー行動に対して 1 つのユースケースを基本にする
- 入力、出力、副作用を明確にする
- 必要な依存をインターフェースとして定義する

### 3. インフラ

- リポジトリ、外部 API、ストレージアクセスを実装する
- クエリや SDK の都合はここに閉じ込める
- セキュリティ上危険な操作は標準化する

### 4. API 境界

- 入力を検証する
- 認証、認可を明示する
- ユースケースの結果を API 契約へ変換する

## レビュー観点

- ルートやハンドラが業務判断を持っていないか
- ユースケースが技術詳細へ依存していないか
- 例外とエラー応答の責務が曖昧でないか
