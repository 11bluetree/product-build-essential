# Domain And Data Modeling

## モデルを分ける

プロダクトでは少なくとも次の 4 つを分けて扱う。

1. ドメインモデル
2. API 契約
3. 永続化モデル
4. UI モデル

同じ項目名を持っていても、責務が違えば別物として扱う。

## ドメインモデルの指針

- エンティティは同一性を持つ
- 値オブジェクトは値の等価性で扱う
- 集約は整合性を保つ単位として設計する
- 生成時と更新時に不変条件を守る

## データ設計の指針

- ID は一貫した方針で生成する
- 状態は列挙可能な値として明示する
- 永続化に便利だからという理由で業務概念を壊さない
- 環境依存の値は保存せず、必要時に組み立てる

## 共有契約の指針

- クライアントとサーバで共有するのは型、スキーマ、DTO に留める
- ドメインロジックやフレームワーク固有型は共有しない
- スキーマから型を導出できる仕組みを優先する

## よくある誤り

- API レスポンスの形をそのままドメインモデルにする
- 画面フォームの都合をそのまま保存モデルへ流し込む
- URL や完全パスなど環境依存の値をデータベースに固定保存する
