# Code Quality And Workflow

## 品質管理の基準

- 型チェックを継続的に通す
- 静的解析を継続的に通す
- フォーマット方針を一貫させる
- 共有設定は 1 か所で管理する

## ワークフロー

1. 実装前に対象の責務を確認する
2. 変更に対するテストを追加または更新する
3. 実装する
4. 型チェック、静的解析、テストを回す
5. 判断理由が残っていない場合は文書化する

## モノレポ運用の原則

- 共有物は明示的なパッケージとして切り出す
- 依存方向を固定する
- 暗黙依存を許さない
- 共通設定は中央管理する

## 自動化

- コミット前フックで軽量な検査を行う
- CI では型、静的解析、テストを必須化する
- デプロイ条件は主幹ブランチへの統合や承認ルールと結びつける
