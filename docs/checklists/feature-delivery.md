# Feature Delivery Checklist

## 仕様

- 変更する業務ルールが明確
- 入出力と失敗条件が明確
- 影響する契約が洗い出されている

## 実装

- ドメインルールが先に定義されている
- 境界の入力検証が追加されている
- 認証、認可要件が反映されている
- 永続化や外部サービスの責務が分離されている

## 品質

- 単体テストを更新した
- 統合テストを更新した
- 型チェックを通した
- 静的解析を通した

## 変更管理

- 互換性影響を確認した
- 新規生成物やローカル専用ファイルを追加した場合は `.gitignore` を更新した
- 必要なら ADR または設計メモを更新した
- 運用上の注意点を共有した
